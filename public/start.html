Quick Start

Last updated: Nov 10th, 2016

Starting the conversation

Entrypoint URL

This is the URL with wich you will be comunicating with


http://rmatch.herokuapp.com/topics/start
Required parameters

These parameters are all sent within a hash named rmatch for an example of the formed(unencoded) URL please skip ahead to example


Name Type Description Default
topic_name string This parameter is the topic for which you will be running rmatch, as rmatch can be used to match within many diffrent topics it is a required parameter
session_data array This is your conversation record data structure, to initiate a new conversation please send the default value [[],[],[]]
Parameter structure
rmatch={"topic_name":"Politk","session_data":[[],[],[]]}
Sample request
http://rmatch.herokuapp.com/topics/start?rmatch={"topic_name":"Politk","session_data":[[],[],[]]}
Sample Response
{
 	"session_data":[[],[],[]],
	"Type":"question",
	"response":[{
				"id":8,
				"question":"¿Debe el gobierno aumentar o disminuir la cantidad de visas de trabajo temporales otorgados a trabajadores inmigrantes altamente cualificados?",
				"topic_id":1,
				"answers":["Disminuir","Mantenerse","Aumentar"]
				}]
}

Continuing the conversation

Entrypoint URL

This is the URL with wich you will be comunicating with once you've gotten your first response for the API


http://rmatch.herokuapp.com/topics/start
Required parameters

These parameters are all sent within a hash named rmatch and are a little diffrent from the ones you send on first contact, for an example of the formed(unencoded) URL please skip ahead to example


Name Type Description Default
topic_name string This parameter is the topic for which you will be running rmatch, as rmatch can be used to match within many diffrent topics it is a required parameter
question_id string This is the ID that was previously given to you in the response
answer_input string This is the index of the answer array given to you in the response (starting at 1)
importance string This is how important the question is to the end user, where 1 is the most important and 5 is the least important
session_data array This is the conversation record data structure, to continue the conversation send the value that you got in the most recent response.
Parameter structure
rmatch={"topic_name":"Politk","question_id":8,"answer_input":2,"importance":1,"session_data":[[],[],[]]}
Sample request
https://rmatch.herokuapp.com/topics/start?rmatch={"topic_name":"Politk","question_id":8,"answer_input":2,"importance":1,"session_data":[[],[],[]]}
Sample Response
{
"session_data":[[8],[2],[1]],
"Type":"question",
"response":[{
			"id":11,
			"question":"¿Deberían los EE.UU. derrocar al presidente Assad de Siria?",
			"topic_id":1,
			"answers":["no","si"]
			}]
}
